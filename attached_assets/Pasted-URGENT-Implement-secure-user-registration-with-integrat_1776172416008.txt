B. Health-System Security Requirements
==================================================

5. Protect sensitive user data
Because this is a health system:
- store passwords securely using proper password hashing
- do not store plaintext passwords
- protect uploaded KYC files/documents
- restrict access to KYC files to authorized admins only
- do not expose sensitive user details unnecessarily in the UI
- avoid leaking sensitive information in logs, errors, or URLs

6. Access control and least privilege
- only authorized roles should review KYC submissions
- only authorized roles should approve, reject, disable, or re-enable users
- ordinary users must not access other users’ KYC records
- role permissions must be enforced in both frontend and backend/API

7. Secure account lifecycle
- Pending users must have limited access
- Verified users can access the system according to RBAC
- Rejected users should be blocked or restricted with a clear message
- Disabled users must not be able to log in

8. Sensitive data handling
- KYC documents should not be publicly accessible
- uploaded files should use protected storage/access control
- do not display full sensitive details unless necessary for authorized review
- keep audit logs for security-sensitive actions

9. Audit logging
Log critical actions such as:
- registration submission
- KYC approval
- KYC rejection
- account disable/enable
- admin review actions

Audit logs should include:
- acting user
- target user
- action
- timestamp

Do not log raw passwords or overly sensitive secrets.

10. Account Status Types

- Pending

- Disabled

11. Admin Verification Panel

- Disabled -> block login with clear message such as "Your account has been disabled. Please contact the administrator."
