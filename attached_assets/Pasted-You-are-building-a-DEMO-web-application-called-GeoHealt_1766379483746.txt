HEALTH LOGIC (CLIENT-SIDE IN src/lib/healthLogic.js)
────────────────────────────────────────

1) Prenatal TT logic
- If tt1Date missing -> Next: Tetanus Shot 1 (TT1) due immediately
- TT2 due = TT1 + 28 days
- TT3 due = TT2 + 180 days
- Status: overdue / due_soon (<=7 days) / upcoming / complete
Use a fixed TODAY date for demo stability: 2025-12-22.
Display as plain language.

Also include Prenatal schedule logic:
- Mother has nextPrenatalCheckDate (static field). Determine status (overdue/due soon/upcoming).

2) Child logic
- Determine next vaccine due from DOB schedule:
  BCG & HepB day 0, Penta1/OPV1 day 60, Penta2/OPV2 day 120, Penta3/OPV3 day 180, MR1 day 270
- Next actionable vaccine is first not done sorted by due date; compute status (overdue/due soon/upcoming/done)
- Child next visit must show:
  - Next vaccine due label
  - NextVisitDate (static field) which is also the next weigh-in date
- Growth “Underweight Risk (Demo)” heuristic based on age months and last weight (simple rule), and clearly label demo + TODO replace with WHO z-score.

3) Senior logic
- HTN meds pickup due status based on nextPickupDate (static)
- Show “meds ready for pickup” if htnMedsReady true and pickedUp false.
- Show medication details: lastMedicationName, doseMg, quantity, lastMedicationGivenDate.

4) Inventory logic
- For each barangay compute:
  - Vaccine status per type: Out / Low / Available / Surplus
  - HTN meds status: Out/Low/Available/Surplus
- Provide a simple “Referral suggestion”:
  If selected barangay is Out for a vaccine/med, recommend top 1–2 barangays with Available/Surplus (static search).

5) Global Notifications (feeds Notification Drawer)
Generate list items from:
- Mothers: TT overdue/due soon; Prenatal check overdue/due soon
- Children: overdue vaccines; due soon vaccines; underweight risk; missing growth monitoring (if no growth in >60 days)
- Seniors: meds pickup due soon/overdue; meds ready for pickup
- Inventory: stock-out alerts (barangay + item)
Prioritize by severity (Overdue first).
