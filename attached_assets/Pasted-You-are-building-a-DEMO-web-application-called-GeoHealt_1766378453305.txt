You are building a DEMO web application called:

────────────────────────────────────────
CRITICAL UX REQUIREMENT (MUST FOLLOW)
────────────────────────────────────────
The end users are technologically handicapped / low digital literacy.
Follow these 10 UI Design Rules strictly and reflect them in the UI structure, labels, and components:

UI RULE 1: One primary goal per screen
- Each page must focus on a single purpose (worklist OR dashboard OR registry), not mixed.

UI RULE 2: Action-first design
- Default view per module should be the WORKLIST (Due Today / Due Soon / Overdue).
- Dashboards are secondary.

UI RULE 3: Plain language, minimal acronyms
- Use “Tetanus Shot 1/2/3” with small text “(TT1/TT2/TT3)” as optional.
- Use “Overdue Vaccines” instead of “Defaulters”.
- Use “Underweight Risk (Demo)” not technical z-scores.

UI RULE 4: Shallow navigation (max 2 levels)
- Module → (Worklist / Dashboard / Registry / Reports)
- No deep nesting.

UI RULE 5: Consistent layout across all pages
- Topbar always same.
- Page sections always: (1) 3–4 large KPI cards (optional) then (2) ONE main table (worklist) then (3) details drawer.

UI RULE 6: Color only for status (3 statuses)
- Red = OVERDUE
- Orange = DUE SOON
- Blue/Green = AVAILABLE/COMPLETED
Always pair color with text labels (never color-only).

UI RULE 7: Large tap targets + readable typography
- Buttons ≥ 44px height.
- Base font 14–16px.
- Entire table row clickable for “View details”.

UI RULE 8: Progressive disclosure
- Show only essential columns; put details in a right-side drawer/modal (“Details”).
- Avoid showing too many vaccine columns in the main list; show them in details view.

UI RULE 9: Error prevention + confirmation
- Actions like “Mark Done”, “Mark Picked Up”, “Record Given” must confirm in a modal.
- Show success banners (“Saved”).

UI RULE 10: Forgiving UX
- Provide undo option for last action (demo state).
- Clear empty states (“No overdue items today”).
- Persist filter selections and last page in localStorage (demo-friendly).

HEALTH LOGIC (DEMO RULES — CLIENT-SIDE)
────────────────────────────────────────

Prenatal – Anti-Tetanus (TT):
- If TT1 missing → Tetanus Shot 1 due immediately
- TT2 due = TT1 + 28 days
- TT3 due = TT2 + 180 days
- Status: overdue (past), due_soon (≤7 days), upcoming, complete
Display labels as:
- “Tetanus Shot 1 (TT1)”, “Tetanus Shot 2 (TT2)”, “Tetanus Shot 3 (TT3)”

Child Vaccination:
- Due dates from DOB:
  - BCG, HepB: day 0
  - Penta1 / OPV1: day 60
  - Penta2 / OPV2: day 120
  - Penta3 / OPV3: day 180
  - MR1: day 270
- Determine next actionable vaccine
- Status: overdue, due_soon, upcoming, done
UI labels:
- “Overdue Vaccines” not “Defaulters”

Nutrition (IMPORTANT DISCLAIMER):

Senior Care:

Inventory:
