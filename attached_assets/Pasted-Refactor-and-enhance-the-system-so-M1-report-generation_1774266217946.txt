==================================================
A. Core Architecture (M1 Data Model)
==================================================

Goal
Make M1 reporting accurate by using:
- a unified patient identity layer (Master Patient Index)
- section-specific program/service records

1. Unified Patient Identity Layer
Create or improve a shared patient index that links:
- Mother profiles
- Child profiles
- Senior profiles
- General/Other patients

This should support:
- global patient search
- deduplication
- linking across modules

2. Do NOT rely on a single “All Patients” table alone
Do not solve M1 by only combining all patients into one flat table.
Instead:
- keep patient identity separate
- use program/service records for reporting

==================================================
B. Section-Based M1 Data Capture
==================================================

3. Add structured records for each M1 section
Each M1 section must have proper data capture.

Example: Family Planning / Modern FP Methods
Capture:
- patient_id
- barangay
- DOB
- age (derived)
- age group (auto-calculated)
- FP method
- status:
  - current user
  - acceptor
  - dropout
- date started
- date stopped (if applicable)
- reporting period relevance

Repeat this pattern for:
- Maternal Care
- Child Care
- Immunization
- Nutrition
- NCD
- Infectious Disease
- Vital Statistics
- all sections from the M1 report

4. Age Group Calculation
- use DOB from patient profile
- use service/program date
- auto-calculate age group
- avoid manual age group entry unless override is needed

C. UI Alignment with M1
==================================================

5. Ensure UI captures all M1-required data

- if missing, add it to the correct module or form

6. Hybrid M1 workflow
- auto-fill where system data exists

- allow manual correction where needed

==================================================
