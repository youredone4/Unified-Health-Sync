These files may:

M1 Import Feature Requirements

1. Multi-file Upload
- Allow selecting multiple files at once
- Display file list before processing
- Allow mixing barangay files, comprehensive reports, and pharmacy files

2. File-Aware Parsing
The system must intelligently handle different file formats.

Implement:
- automatic column detection
- flexible mapping
- sheet detection for Excel files

Common mappings:
- barangay, brgy, barangay_name -> barangay
- disease, disease_name, category -> disease_name
- cases, total, count -> cases
- date, reporting_date, month -> reporting_date

If a required column is not found:
- attempt best-match detection
- otherwise flag as validation error

3. Barangay Detection Logic
Determine barangay using:
- file name (e.g., AMOSLOG.xls -> barangay = AMOSLOG)
- column values inside the file
- mapped barangay field

If a file contains multiple barangays:
- preserve them correctly

4. Multi-Sheet Handling
- detect available sheets
- process relevant sheets automatically
- ignore empty or irrelevant sheets

5. Data Normalization
Convert all extracted data into a unified structure:

barangay | disease_name | cases | reporting_date

Ensure:
- consistent formatting
- numeric validation for cases
- clean string values

6. Data Aggregation
- merge all uploaded files into one dataset
- group by barangay and disease_name
- sum case counts

7. Preview Before Import
Before saving:
- show combined preview table
- show total rows
- show detected barangays
- show detected diseases
- allow user confirmation

8. Validation Summary
Display:
- Total rows
- Valid rows
- Error rows

For invalid rows, show reasons such as:
- missing barangay
- missing disease_name
- invalid case value
- unmapped columns
- corrupted row

Display error rows in a separate table if possible.

9. Import Options
Provide:
- Append Data
- Replace Existing Data

Behavior:
- Append = add to existing M1 dataset
- Replace = overwrite relevant M1 data

Require confirmation for replace.

10. Import History Log
Track:
- date/time
- user
- uploaded file names
- number of rows processed
- valid rows imported
- error count
- import mode (append or replace)

Display as a table, latest first.

11. Download Templates
Provide:
- CSV template
- Excel template

Structure:
barangay,disease_name,cases,reporting_date

12. Role-Based Access
Allow only:
- System Admin
- Municipal Health Officer
- Senior Health Admin

Do NOT allow:
- Team Leader (Barangay Nurse)

Use the existing role system. Do not duplicate role logic.

13. Integration
Imported data must reflect immediately in:
- M1 reports
- dashboards
- analytics
- summary cards

Do NOT break existing calculations.

14. Performance
- handle multiple Excel files efficiently
- avoid UI freezing
- process large files safely

15. Table Behavior
For:
- preview table
- validation error table
- import history log

Use:
- 10 rows per page by default
- pagination enabled
- search if available
