==================================================
A. Create three scripts under server/
==================================================

1) server/reset-test-data.ts — Truncate-only reset
   TRUNCATE ... RESTART IDENTITY CASCADE on these tables, in order:
     Visits:        prenatal_visits, child_visits
     Registries:    mothers, children
     Clinical:      consults, disease_cases, tb_patients,
                    fp_service_records, death_events
     Inventory:     inventory, medicine_inventory
     M1 instances:  m1_indicator_values, m1_report_header, m1_report_instances
     Messaging:     sms_outbox, messages, conversations,
                    global_chat_messages, direct_messages
   Preserve (do NOT touch):
     DSWD data:     seniors, senior_visits, senior_med_claims
     Auth/config:   users, sessions, barangays, user_barangays,
                    barangay_settings, municipality_settings, theme_settings,
                    health_stations, audit_logs
     M1 lookups:    m1_template_versions, m1_indicator_catalog
   Safety gates (all required):
     - Env var ALLOW_TEST_RESET=true
     - CLI flag --confirm  (also support --dry-run that only prints counts)
     - Optional env PRODUCTION_DB_HOST — refuse if DATABASE_URL host matches
     - Print DATABASE_URL host + db name, wait 5 seconds, then execute
     - Run inside a single transaction; rollback on any error
     - Print before/after row counts and exit non-zero if anything is non-zero

2) server/seed-patients.ts — KEEP EXISTING
   This already seeds mothers, children, disease_cases, tb_patients.
   Do not modify except to confirm it skips seniors (DSWD data is untouched).

3) server/seed-transactional-data.ts — NEW
   Reads existing mothers + children from DB, then generates data distributed
   from 2025-01-01 through the current month:
     - prenatal_visits   : 3–6 per mother, ~4 weeks apart, TT-indicative notes
     - child_visits      : 2–5 per child, distinct months, immunization notes
     - consults          : ~10 per barangay per month (ranges 7–13), mixed
                           consult types, ages, barangays, diagnoses
     - fp_service_records: ~8 per barangay per month (ranges 5–11), mixed
                           FP_METHODS + FP_STATUSES, reportingMonth = YYYY-MM
   Constraints:
     - prenatal_visits has unique(motherId, visitNumber) — honor it
     - child_visits has unique(childId, visitNumber) — honor it
     - fp_service_records.reportingMonth must be YYYY-MM for M1 scoping
     - fp_service_records.fpMethod must be one of FP_METHODS (see schema)
     - fp_service_records.fpStatus must be one of FP_STATUSES
   Safety gates:
     - ALLOW_TEST_RESET=true AND --confirm required
     - Refuses if prenatal_visits/child_visits/consults/fp_service_records
       already have rows, unless --force is passed
   Bulk-insert in chunks of 500 for performance.

ALLOW_TEST_RESET=true npx tsx server/seed-transactional-data.ts --confirm
